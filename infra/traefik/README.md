# Traefik — Management VM Setup

Central Traefik instance on the management VM handles TLS termination and
reverse-proxying for all tenant services via IONOS DNS ACME challenge.

## Architecture

```
Internet ──► :80  ──► redirect ──► :443 (websecure)
                                      │
                        IONOS DNS ACME (wildcard cert)
                                      │
                    /etc/traefik/dynamic/*.yml
                        (auto-reloaded, no restart)
                                      │
                  ┌───────────────────┼───────────────────┐
             default.yml        kunde-a.yml          kunde-b.yml
           (dashboard)       (app1, app2...)        (app1...)
```

## Required Environment Variables

| Variable       | Description                                               |
|----------------|-----------------------------------------------------------|
| `IONOS_API_KEY` | IONOS DNS API key in format `<public-prefix>.<secret>`. Create at [developer.hosting.ionos.de](https://developer.hosting.ionos.de/) |
| `ACME_EMAIL`   | Email for Let's Encrypt notifications                     |

## Deploy on Management VM

### Quick start (Ansible — recommended)

```bash
cd automation/ansible
ansible-playbook deploy-traefik.yml \
  -e acme_email=ops@irongeeks.eu \
  -e ionos_api_key=<public>.<secret>
```

### Manual deploy

```bash
# 1. Create directories
sudo mkdir -p /etc/traefik/{dynamic,acme}
sudo chmod 600 /etc/traefik/acme   # acme.json must be 600

# 2. Copy static config
sudo cp infra/traefik/traefik.yml /etc/traefik/traefik.yml

# 3. Copy default dynamic config
sudo cp infra/traefik/dynamic/default.yml /etc/traefik/dynamic/default.yml

# 4. Set environment and run Traefik
export IONOS_API_KEY="<public-prefix>.<secret>"
export ACME_EMAIL="ops@irongeeks.eu"

docker run -d \
  --name traefik \
  --restart unless-stopped \
  -p 80:80 -p 443:443 \
  -e IONOS_API_KEY \
  -e ACME_EMAIL \
  -v /etc/traefik:/etc/traefik \
  -v /var/run/docker.sock:/var/run/docker.sock:ro \
  traefik:v3
```

## Per-Tenant Dynamic Configs

Each tenant gets a YAML file in `/etc/traefik/dynamic/`:

```
/etc/traefik/dynamic/
  default.yml       ← always present (dashboard, redirect)
  kunde-a.yml       ← generated by provision-tenant.yml
  kunde-b.yml
  ...
```

Traefik watches this directory and reloads without restart when files change.

The webapp generates tenant configs at:
`GET /api/tenants/:id/traefik-config` (admin-only)

The Ansible provision-tenant playbook renders and deploys configs automatically.

## ACME / IONOS DNS Notes

- Traefik uses [LEGO](https://go-acme.github.io/lego/dns/ionos/) for DNS challenge
- `IONOS_API_KEY` format: `<public-prefix>.<secret>` — get from IONOS Developer Portal
- Certificates are stored in `/etc/traefik/acme/acme.json` (must be mode `600`)
- DNS propagation can take 30–120 seconds on first certificate issuance

## Security Notes

- Traefik dashboard is restricted to internal IP ranges via `mgmt-ipallowlist`
- Adjust source ranges in `dynamic/default.yml` to match your management VLAN
- Never expose `insecure: true` in production
